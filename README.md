# RaidCoder

**AtCoder のレイド版。** AI が毎週ランク S〜E のアルゴリズム問題を生成し、みんなの AC がボスへのダメージになる**非同期協力型・競技プログラミングゲーム**。

> 月曜 00:00、新たなボスが出現する。HP は数十万。
> 誰かが AC するたび、全員の画面で HP バーがリアルタイムに減っていく。
> 週末までに削り切れば全員に報酬。週間ダメージでレートも動く。

- 毎週月曜(JST)にボスと問題6問(S / A / B / C / D / E)を AI が自動生成
- AC するとランク・速度・先制に応じたダメージ。ボス HP はライブ更新(Supabase Realtime)
- 週間ダメージで順位がつき、AtCoder 風レーティングが変動
- 掲示板でアイデア相談 OK(協力プレイ)。他人のコードは自分が AC した問題 or 週終了後に閲覧可
- 解説と公式解は**1週間後(週終了時)に公開**

対応言語: Python / Rust / TypeScript / Java(標準入出力形式、AtCoder と同じ)

## ルール

### ダメージ計算

```
ダメージ = floor( 基礎ダメージ × 速度倍率 × 先制ボーナス )
```

| ランク | E | D | C | B | A | S |
|---|---|---|---|---|---|---|
| 基礎ダメージ | 500 | 1000 | 2000 | 3500 | 5500 | 8500 |

- **速度倍率** — 週の序盤ほど強い。週開始直後 **×1.3** → 週末 **×1.0** へ線形に減衰(`1 + 0.3 × (1 − 週の経過割合)`)
- **先制ボーナス(First Blood)** — その問題を全プレイヤーで最初に AC すると **×1.5**
- ダメージが入るのは**各問題につき自分の最初の AC だけ**。再提出・週終了後の AC は練習扱い(ダメージ 0)

### ボスと報酬

- ボスの HP = `20000 × アクティブプレイヤー数`(直近14日に提出した人数、最低3)— 人が増えるほどボスも強くなる
- 週末までに HP を 0 にできれば、**参加者全員(週間ダメージ > 0)に +2000 EXP**
- 撃破後の AC もランキングには全額計上される。オーバーキル歓迎
- AC したダメージはそのまま EXP としても蓄積

### レーティング

- 初期レート **1000**。週間ダメージの順位でパフォーマンス(perf)が決まる:
  - `perf = 1200 + 800 × (N − 順位) / (N − 1)`(N = 参加者数。1位 = 2000、最下位 = 1200)
  - `新レート = 旧レート + (perf − 旧レート) / 4`
- 参加するだけ(1問でも AC)で perf 1200 以上が保証される。**継続は力**
- レート色(AtCoder 風):

  | 2000+ | 1800+ | 1600+ | 1400+ | 1200+ | 1000+ | 800+ | それ未満 |
  |---|---|---|---|---|---|---|---|
  | 赤 | 橙 | 黄 | 青 | 水 | 緑 | 茶 | 灰 |

### 情報公開のタイミング(ネタバレ防止、RLS で強制)

| | 週の途中 | 週終了後 |
|---|---|---|
| 問題文・サンプルケース | ○ | ○ |
| 解説・公式解 | × | ○ |
| 他人の提出コード | 自分が AC 済みの問題のみ ○ | ○ |
| 掲示板(作戦会議) | ○ いつでも歓迎 | ○ |

**「解答は1週間後」**: 週が終了した瞬間に、解説・公式解・全員の提出コードが自動公開される。

## 開発クイックスタート

前提: Node.js 24 / Docker。詳細は [infra/LOCAL_DEV.md](./infra/LOCAL_DEV.md)。

```bash
# 1. DB + Auth(migration とチュートリアル週の seed が自動適用される)
npx supabase start

# 2. web — .env.local を作って起動(→ http://localhost:3000)
cd web && npm install && npm run dev

# 3. ジャッジ — macOS では Judge0 が動かないためローカル実行モードで(別ターミナル)
cd judge-bridge && npm install && npm run dev   # .env に EXECUTOR=local
```

本番デプロイ(Vercel + Supabase + Lightsail、月額 ¥5,000 以内)は [infra/DEPLOY.md](./infra/DEPLOY.md)。

## 既知の制約

- **Judge0 CE v1.13.1 の言語バージョンは古い**: Python 3.8 / Rust 1.40 / TypeScript 3.7 / Java (OpenJDK 13)。Python の `match` 文など新しい構文は使えない
- Judge0 は **cgroup v1 の privileged Linux が必須**(ホストの GRUB 設定が必要。[DEPLOY.md](./infra/DEPLOY.md) 参照)。macOS では動かないため、ローカル開発は judge-bridge の `EXECUTOR=local`(サンドボックスなし・開発専用)を使う
- 問題は AI 生成。**公式解が全テストケースを通ることは配信前に検証済み**だが、難易度や問題文の質は週によってブレることがある
- 無料枠前提の構成(Vercel Hobby / Supabase Free)のため、大人数(数百人規模)になったら要プラン見直し
