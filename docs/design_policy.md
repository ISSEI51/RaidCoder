# デザイン方針

## 基本方針
デザインには外部MCPやスキルを使用して統一感とクオリティのレベルを上げる

## UI
1. shadcn/ui MCP
claudeが作る毎回異なるボタン、モーダル、カードを作る問題を解決する。
設定方法
```bash
pnpm dlx shadcn@latest mcp init --client claude
```
※shadcnのみだと普通のSaasの管理画面になりやすい

2. Chrome DevTools MCP
実際の画面をclaude自身に評価させる
- スクリーンショット取得
- レスポンシブ確認
- コンソールエラー確認
- DOMやCSSの調査
- パフォーマンス確認
設定方法
```sh
claude mcp add chrome-devtools \
  --scope user \
  npx chrome-devtools-mcp@latest
```

3. Storybook MCP
claudeが既存コンポーネントと利用ルールを理解し、Storyを生成してインタラクションテストやアクセシビリティチェックを実行できる
設定方法
```sh
npx storybook add @storybook/addon-mcp
```
最初から導入ではなく、UIの方向性が固まってからで十分

4. Replicate MCP
後述するキャラクター画像生成に使用する
設定方法
```sh
claude mcp add replicate \
  https://mcp.replicate.com/sse \
  --transport sse \
  --scope user
```
claudeには次のように作業させる
```
RaidCoder用キャラクターを作成してください。

最初に全キャラクターを生成せず、
1. アートディレクション案
2. 基準キャラクター1体
3. キャラクターシート
4. 残りのキャラクター
の順で進めてください。

基準キャラクターが承認されるまで、
他のキャラクターは生成しないでください。

制約:
- 絵文字を使用しない
- 512×512px
- 胸上のポートレート
- 背景透過
- 全員で輪郭線、陰影、頭身、光源を統一
- 近未来の競技プログラミングゲーム
- 安っぽいネオンや過剰な発光は避ける
- 既存のゲームやアニメキャラクターに似せない
```
最初に1体を完成させ、その画像を参照画像として残りを作らせる
とにかく統一感を重視

## キャラクター
キャラクターを絵文字に置き換えるのは禁止
以下の条件でキャラクターを作成する
- キャラクター数: 6~10体
- 表示: 胸上の正方形ポートレート
- サイズ: 512*512px
- 背景: 透過または同一形式の円形背景
- 世界観: 近未来のプログラミング競技/サイバー競技場
- 色: キャラクターごとにアクセントカラーを1色
- 画風: アニメによりすぎない、ゲームカード風のセミフラットイラスト
例えば、普通の「剣士・魔法使い」ではなく競プロ向けの役割にする
| キャラクター      | モチーフ        |
| ----------- | ----------- |
| Pathfinder  | グラフ探索・迷路    |
| Architect   | 動的計画法・設計    |
| Hacker      | ビット演算・XOR   |
| Sentinel    | セキュリティ・検証   |
| Speedrunner | 貪欲法・高速実装    |
| Oracle      | 数学・確率       |
| Debugger    | バグ発見・テスト    |
| Commander   | チーム戦・レーティング |



## CLAUDE.mdに固定するべきルール
```markdown
## UI Design Rules

- UIアイコンとして絵文字を使用しない
- アイコンは lucide-react に統一する
- ダークテーマを基本とする
- SaaS管理画面ではなく、競技・eスポーツの雰囲気にする
- 色は背景色、サーフェス色、アクセント色、危険色の4系統に制限する
- グラデーションと発光表現を乱用しない
- すべてをカードで囲まない
- 主要操作は1画面につき1つに絞る
- 角丸は基本8px。ピル型ボタンを乱用しない
- 問題文とコードエディタの可読性を装飾より優先する
- 見出し、本文、補助テキストの階層を明確にする
- 実装後は必ずブラウザでスクリーンショットを確認する
```
